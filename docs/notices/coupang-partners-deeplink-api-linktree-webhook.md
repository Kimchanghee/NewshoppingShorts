# 쿠팡 파트너스 딥링크 API와 Linktree 웹훅 자동 발행 안내

이 글은 설정 > 연결 도우미에서 보이는 `쿠팡 파트너스 딥링크`, `Linktree 웹훅 자동 발행` 항목이 왜 있는지 설명합니다.

## 쿠팡 파트너스 딥링크 API가 필요한 이유

쿠팡 파트너스 딥링크 API는 원본 `coupang.com` 상품 URL을 파트너스 추적 링크로 바꾸는 기능입니다. API를 연결하면 YouTube 설명, 고정 댓글, Linktree 카드에 들어가는 구매 링크를 파트너스 정산 추적이 가능한 링크로 자동 변환할 수 있습니다.

딥링크 API는 영상 제작 자체를 위해 필요한 기능이 아닙니다. API 키가 없어도 SSMaker는 원본 쿠팡 URL로 영상 제작과 업로드 큐를 계속 진행합니다. 단, 클릭 추적과 파트너스 정산을 자동 링크에 정확히 남기려면 Access Key와 Secret Key가 필요합니다.

## 언제 필요한가요?

다음 상황에서는 딥링크 API 연결을 권장합니다.

- 쿠팡 상품 링크를 파트너스 수익 추적 링크로 자동 변환하고 싶을 때
- 대량 큐에서 원본 상품 URL을 매번 수동으로 파트너스 링크로 바꾸기 어렵게 됐을 때
- YouTube 설명, 고정 댓글, Linktree 카드에 같은 구매 링크를 일관되게 넣고 싶을 때
- 추후 어떤 상품 링크가 자동 발행됐는지 기록하고 정산 추적 기준을 맞추고 싶을 때

다음 상황에서는 없어도 됩니다.

- 지금은 영상 생성과 YouTube 업로드만 먼저 확인하려는 경우
- 쿠팡 파트너스 최종 승인이 아직 안 된 경우
- Linktree 공개 주소만 연결해두고, 링크 카드는 수동으로 추가하려는 경우
- 자동화가 원본 쿠팡 URL로 진행돼도 당장 문제가 없는 경우

## 언제 얻을 수 있나요?

쿠팡 파트너스 API 안내에 따르면 파트너스 API는 일반적으로 최종 승인된 쿠팡 파트너스 회원에게 제공되며, Access Key와 Secret Key는 쿠팡 파트너스의 `Tools > 파트너스 API` 메뉴에서 발급받습니다. 최종 승인 상태인데도 메뉴가 보이지 않거나 키 생성이 막혀 있으면 쿠팡 파트너스 안내에 따라 계정 상태를 확인해야 합니다.

발급된 Access Key와 Secret Key는 비밀번호처럼 취급해야 합니다. 화면 공유, 로그, 문의 메시지에 그대로 노출하지 마세요.

## Linktree 웹훅 자동 발행은 무엇인가요?

여기서 말하는 웹훅은 Linktree 계정 비밀번호나 단순 API 키가 아닙니다. 사용자가 Make, Zapier, n8n 같은 자동화 서비스에서 만든 `받는 주소`입니다. SSMaker가 새 쿠팡 링크를 만들면 이 주소로 상품 제목과 구매 링크 정보를 보내고, 자동화 서비스가 Linktree 관리자 계정에 새 링크 카드를 추가하는 방식입니다.

SSMaker가 웹훅으로 보내는 핵심 데이터는 다음 형태입니다.

```json
{
  "title": "상품 제목",
  "url": "구매 링크",
  "description": "짧은 설명",
  "source_url": "원본 쿠팡 URL",
  "platform": "linktree"
}
```

웹훅 자동 발행은 선택 기능입니다. Webhook URL이 없으면 `쿠팡 링크 생성 시 Linktree 웹훅으로 자동 발행`을 꺼두고, 공개 Linktree 주소만 저장하거나 Linktree 관리자 화면에서 수동으로 카드를 추가하면 됩니다.

## 권장 설정 순서

1. Linktree 공개 프로필 URL을 먼저 저장합니다.
2. 쿠팡 파트너스 최종 승인 후 Access Key와 Secret Key를 입력합니다.
3. Linktree 카드를 자동으로 추가하고 싶으면 Make, Zapier, n8n 중 하나로 Webhook URL을 만듭니다.
4. `테스트 발행`으로 웹훅이 Linktree 카드 추가까지 연결되는지 확인합니다.
5. 테스트가 끝난 뒤 풀자동화 큐에서 실제 링크를 예약 실행합니다.

처음 세팅할 때는 딥링크 API와 웹훅을 모두 한 번에 켤 필요가 없습니다. 영상 생성과 업로드가 먼저 안정적으로 되는지 확인한 뒤, 파트너스 추적과 Linktree 카드 자동 발행을 순서대로 붙이는 방식이 가장 안전합니다.
